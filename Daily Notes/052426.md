## Resolved: ChargeMode ↔ InstallationType — controlli distinti
Probe live + doc V2C ufficiali confermano che `ChargeMode` (LAN keyword) e `InstallationType` (cloud `/device/inst_type`) sono **due controlli distinti**, non sinonimi.
- LAN ChargeMode (doc Datamanager rev 04/05/26): 0=Monophasic, 1=Threephasic, **2=Mixed**
- Cloud /device/inst_type (doc OpenAPI cloud, "Set Installation Type"): 0=Monophase, 1=Three-phase, **2=Photovoltaic**

Il valore 2 ha semantica diversa. Live stato divergente stabile (LAN=1, cloud=2) coerente con l'interpretazione "axes ortogonali": tipo impianto fisico vs modo di carica corrente. Codice attuale già corretto: due Select separate, una via cloud setter (`async_set_installation_type`) e l'altra via LAN router (`cloud_call=None`). Nessun fix necessario. Probe distruttiva LAN→cloud cancellata perché la doc dirime il dubbio.
